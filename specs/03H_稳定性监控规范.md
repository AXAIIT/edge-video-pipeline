# 03H_稳定性监控规范

## 适用范围

本规范用于执行项目三的“边缘设备稳定性与监控系统”实验，验证视频 pipeline 是否能持续运行。

这是从原独立项目合并进项目三的横向能力，不能只在总规范里提到，必须真实执行并写入 `projects/03_video_pipeline/reports/stability_report.md`。

## 执行原则

下面的 sustained 时长、监控字段和脚本接口是推荐路径，不是唯一实现。可以替换监控工具、采样脚本、资源采集方式或输入源，但不能降低证据要求：必须保留连续运行时长、资源曲线、队列长度、丢帧率、异常恢复、日志路径和问题记录。替代路径必须写入对应 `projects/03_video_pipeline/runs/`，并同步更新 `projects/03_video_pipeline/reports/stability_report.md`。

## 路径口径

本 spec 使用当前仓库结构：

- pipeline 配置放在 `projects/03_video_pipeline/configs/pipeline/`。
- 固定长视频放在 `data/videos/video_fixed_v1/`。
- raw result 放在 `benchmark/raw/03_video_pipeline/<target>/`。
- runtime 日志放在 `logs/runtime/03_video_pipeline/<target>/`。
- monitor 日志放在 `logs/monitor/03_video_pipeline/<target>/`。
- failure 日志放在 `logs/failures/03_video_pipeline/<target>/`。
- 稳定性报告放在 `projects/03_video_pipeline/reports/stability_report.md`。

## 前置条件

- pipeline 已完成目标后端集成。
- 已建立环境基线。
- 已准备固定输入源和实时输入源。
- 已准备资源监控脚本。
- 已完成至少一次 10 分钟 runtime benchmark。
- run 记录必须对齐 `projects/03_video_pipeline/specs/00_run_record_template.md`。
- 环境基线必须对齐 `projects/03_video_pipeline/specs/00_environment_baseline_template.md`。
- raw result 必须符合 `benchmark/schemas/video_pipeline_raw_schema.yaml`，且每行表示一帧。

## 测试档位

| 档位 | 时长 | 是否必须 | 作用 |
|---|---:|---|---|
| smoke | 10 分钟 | 必须 | 确认配置可跑 |
| short sustained | 30 分钟 | 必须 | 基础稳定性 |
| acceptance sustained | 2 小时 | 项目验收目标 | 作品集核心证据 |
| long sustained | 8 小时 | 条件允许 | 压力验证 |

## 执行步骤

### 1. 建立 run 记录

新建：

```text
projects/03_video_pipeline/runs/<yyyymmdd>_<target>_yolo11n_<backend>_stability_<tier>/run.md
```

run 中必须字段化记录：

```yaml
stability:
  stability_tier: smoke | short_sustained | acceptance_sustained | long_sustained
  target_duration_sec:
  actual_duration_sec:
  completed:
  stop_reason:
  restart_count:
  reconnect_count:
  crash_count:
  watchdog_timeout_count:
```

### 2. 启动资源监控

命令模板：

```bash
RUN_ID=<run_id>
python3 projects/03_video_pipeline/scripts/monitor_device.py \
  --board <jetson_8gb|rk3588_8gb|rdk_x5_8gb> \
  --interval-sec 1 \
  --output logs/monitor/03_video_pipeline/<target>/${RUN_ID}_monitor.csv &
echo $! > logs/monitor/03_video_pipeline/<target>/${RUN_ID}_monitor.pid
```

如果监控脚本是阻塞式执行，必须在 run 中说明；如果后台执行，测试结束后必须停止：

```bash
kill "$(cat logs/monitor/03_video_pipeline/<target>/${RUN_ID}_monitor.pid)"
```

监控至少包含：

- CPU/GPU/NPU/BPU 利用率。
- 内存。
- 温度。
- 功耗或电源模式。
- swap，如果存在。
- 进程状态。

### 3. 运行 sustained 测试

30 分钟：

```bash
build/03_video_pipeline/video_pipeline_app \
  --config projects/03_video_pipeline/configs/pipeline/<target>_<backend>_pipeline.yaml \
  --input-source-id video_set_stability_v1 \
  --input data/videos/stability_playlist_v1.txt \
  --duration-sec 1800 \
  --raw-output benchmark/raw/03_video_pipeline/<target>/<run_id>.jsonl \
  > logs/runtime/03_video_pipeline/<target>/<run_id>.log 2>&1
```

2 小时：

```bash
build/03_video_pipeline/video_pipeline_app \
  --config projects/03_video_pipeline/configs/pipeline/<target>_<backend>_pipeline.yaml \
  --input-source-id video_set_stability_v1 \
  --input data/videos/stability_playlist_v1.txt \
  --duration-sec 7200 \
  --raw-output benchmark/raw/03_video_pipeline/<target>/<run_id>.jsonl \
  > logs/runtime/03_video_pipeline/<target>/<run_id>.log 2>&1
```

### 4. 聚合稳定性指标

命令模板：

```bash
python3 projects/03_video_pipeline/scripts/monitor/analyze_stability.py \
  --raw benchmark/raw/03_video_pipeline/<target>/<run_id>.jsonl \
  --monitor logs/monitor/03_video_pipeline/<target>/<run_id>_monitor.csv \
  --output projects/03_video_pipeline/reports/stability_report.md
```

## 记录要求

- monitor 日志保存到 `logs/monitor/03_video_pipeline/<target>/`。
- raw result 保存到 `benchmark/raw/03_video_pipeline/<target>/`。
- runtime 日志保存到 `logs/runtime/03_video_pipeline/<target>/`。
- 异常日志保存到 `logs/failures/03_video_pipeline/<target>/`。
- 报告写入 `projects/03_video_pipeline/reports/stability_report.md`。
- 如果测试中断，必须保留中断前 raw result 和失败日志。
- 2 小时测试没完成时，必须标记目标未达成，不能用 30 分钟结果替代 2 小时验收。
- 中断状态判定：可复现失败写 `fail`；资源或监控不完整但可运行写 `degraded`；缺硬件/权限写 `blocked`；缺后端证据写 `not_verified`。

## 输出文件

| 文件 | 要求 |
|---|---|
| `benchmark/raw/03_video_pipeline/<target>/*stability*.jsonl` | sustained raw result |
| `logs/monitor/03_video_pipeline/<target>/*stability*.csv` | 资源曲线 |
| `logs/runtime/03_video_pipeline/<target>/*stability*.log` | 运行日志 |
| `projects/03_video_pipeline/reports/stability_report.md` | 稳定性报告 |
| `projects/03_video_pipeline/reports/runtime_benchmark.md` | 引用 sustained 指标 |

## 验收标准

- 30 分钟测试必须通过。
- 2 小时测试作为项目验收目标。
- 内存不持续增长到不可用。
- 队列长度不持续增长到不可控。
- 温度、降频、异常恢复有记录。
- 报告包含 MLPerf-style Summary。

## 降级和问题库

下面问题必须进入问题库：

- 内存增长。
- 温控降频。
- 断流无法恢复。
- 队列堆积。
- 服务崩溃。
- 监控日志缺失。
